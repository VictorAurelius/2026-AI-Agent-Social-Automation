### As Documentation

Skills provide step-by-step guides and best practices. Read them when:
- Learning a new process
- Setting up a new integration
- Troubleshooting issues
- Standardizing workflows

### As References

Quick reference while working:
- Check file organization rules before creating files
- Review commit message format before committing
- Consult content templates before writing posts
- Reference prompt templates when configuring AI

### With Claude Code

Skills are automatically available to Claude Code AI assistant:
- Claude can reference skills when helping you
- Claude follows conventions defined in skills

- Claude uses proper commit message format

## 🛠️ Creating New Skills

When you discover a repeatable pattern or best practice:

1. Create new skill in `.claude/skills/`
2. Follow skill template format:
   - Purpose & overview
   - When to use
   - Step-by-step guide
   - Examples
3. Add to [SKILLS-INDEX.md](./.claude/skills/SKILLS-INDEX.md)
4. Update this quick reference if it's a core skill
5. Commit with `feat(skills): add [skill-name]`

**Example skill names:**
- `facebook-posting.md` - Facebook posting guidelines
- `image-generation.md` - AI image generation workflow
- `seo-optimization.md` - SEO best practices
